# Reportes y Gráficos

## Descripción

La funcionalidad de **Reportes y Gráficos** en Solop ERP permite a los usuarios generar, visualizar y personalizar reportes operativos con representaciones gráficas integradas. A través de una interfaz dividida en dos paneles, el usuario puede configurar parámetros, aplicar filtros y cambiar el tipo de gráfico para adaptar la presentación de los datos a sus necesidades de análisis.

Esta funcionalidad está disponible en la interfaz moderna (Retails) de Solop ERP.

## ¿Cuándo se utiliza?

Se utiliza cuando el usuario necesita:

- Consultar reportes operativos con datos agrupados por mes, grupos de productos u otras dimensiones.
- Visualizar tendencias mediante gráficos de barras, líneas u otros tipos disponibles.
- Aplicar filtros dinámicos para obtener vistas específicas de la información.
- Preparar información para impresión o exportación.
- Comparar totales generales y subtotales de líneas dentro de un mismo reporte.

## Acceso

El acceso a los reportes se realiza desde el menú principal de Solop ERP:

**Menú Principal → Módulo correspondiente → Reportes**

La funcionalidad está disponible en la interfaz moderna de Solop ERP (Retails). Una vez dentro del módulo, se puede acceder al listado de reportes disponibles para la organización y el rol del usuario.

## Interfaz Principal

La pantalla de reportes presenta un **panel dividido en dos secciones**:

- **Panel izquierdo**: contiene los parámetros de configuración, filtros disponibles y opciones del reporte.
- **Panel derecho**: muestra el resultado generado del reporte seleccionado.

Los campos en la interfaz están alineados para facilitar la lectura, con valores numéricos a la derecha y etiquetas descriptivas a la izquierda.

El usuario puede alternar entre **vista estándar** y **vista compacta** del reporte según la cantidad de datos a visualizar.

## Campos Principales

### Sección de Configuración del Reporte

| Campo | Descripción |
|-------|-------------|
| Modelo de reporte | Selector del tipo o categoría de reporte a generar |
| Reporte específico | Lista de reportes disponibles según el modelo seleccionado |
| Formato de visualización | Modo de presentación: estándar o compacto |
| Parámetros de visualización | Ajustes de presentación propios de la interfaz Retails |

### Sección de Parámetros

| Campo | Descripción |
|-------|-------------|
| Totales generales | Muestra la sumatoria global del reporte |
| Totales por líneas | Muestra subtotales agrupados por cada línea del reporte |
| Opciones de impresión | Configuraciones para el formato impreso del documento |

### Sección de Gráficos

| Campo | Descripción |
|-------|-------------|
| Tipo de gráfico | Selector del tipo de gráfico (barras, líneas, torta, etc.) |
| Agrupación | Criterio de agrupación de los datos en el gráfico (por mes, por grupo, etc.) |

## Acciones Disponibles

| Acción | Descripción |
|--------|-------------|
| Generar reporte | Ejecuta el reporte seleccionado con los parámetros configurados |
| Cambiar parámetros | Modifica los valores de configuración sin abandonar la pantalla |
| Aplicar filtros | Agrega criterios de filtrado para reducir el conjunto de datos mostrado |
| Cambiar tipo de gráfico | Alterna entre los diferentes tipos de representación gráfica disponibles |
| Guardar cambios de gráfico | Persiste la configuración del gráfico modificado |
| Ver cuerpo completo | Expande la vista para mostrar el reporte completo sin paginación |
| Preparar para impresión | Genera una versión del reporte lista para ser impresa |

## Flujo del Proceso

**Paso 1: Acceso a la interfaz**

Ingresar a Solop ERP con las credenciales de usuario y acceder a la interfaz Retails. Desde el menú, navegar al módulo que contiene los reportes requeridos.

**Paso 2: Selección del reporte**

En el panel izquierdo, seleccionar el **modelo de reporte** deseado. A continuación, elegir el **reporte específico** de la lista disponible según las necesidades del usuario.

**Paso 3: Configuración del reporte**

Ajustar las configuraciones disponibles en el panel izquierdo:

- Revisar los formatos de presentación disponibles.
- Configurar parámetros como totales generales, subtotales por líneas y opciones de impresión.

**Paso 4: Generación del reporte**

Hacer clic en el botón de generación del reporte para ejecutarlo con los parámetros configurados. El sistema presentará los resultados en el panel derecho.

**Paso 5: Visualización y verificación**

Revisar el reporte generado para verificar que toda la información esté correcta. Se puede visualizar el cuerpo completo del reporte o navegar entre secciones.

**Paso 6: Ajuste de parámetros (opcional)**

Si el resultado no es el esperado, modificar los parámetros en el panel izquierdo y regenerar el reporte sin necesidad de abandonar la pantalla.

**Paso 7: Revisión para impresión**

Antes de imprimir, revisar los totales de líneas y el gran total del reporte. Una vez verificado, preparar el documento para su impresión.

**Paso 8: Exploración de gráficos asociados**

Acceder a la pestaña o sección de gráficos del reporte. El sistema mostrará por defecto un gráfico de barras con los totales agrupados (por ejemplo, totales por mes o por grupo de productos).

**Paso 9: Modificación del tipo de gráfico (opcional)**

Si el tipo de gráfico predeterminado no es el adecuado, seleccionar otro tipo de representación desde el selector disponible. Guardar los cambios realizados para que persistan.

**Paso 10: Aplicación de filtros**

Utilizar la sección de filtros del panel izquierdo para personalizar los datos presentados. Los filtros permiten visualizar información específica tanto en la tabla de datos como en los gráficos asociados. Los resultados se actualizan en tiempo real al aplicar cada filtro.

## Consideraciones Importantes

- Es necesario contar con los **permisos adecuados** para acceder a los reportes. Los reportes disponibles varían según el rol del usuario en el sistema.
- Verificar siempre que la **información ingresada en los parámetros** sea correcta antes de generar el reporte, para evitar resultados incorrectos.
- Los **gráficos se presentan en modo barra por defecto**. El tipo de gráfico puede cambiarse según las necesidades de presentación, y los cambios pueden guardarse para uso futuro.
- La **vista compacta** es útil cuando se trabaja con reportes que contienen una gran cantidad de columnas o registros.
- Los **filtros dinámicos** permiten reducir el conjunto de datos sin necesidad de regenerar completamente el reporte desde cero.
- Familiarizarse con los **atajos de teclado** de la interfaz puede agilizar la navegación entre secciones del reporte.
- Guardar los reportes generados en una **ubicación organizada** facilita el acceso posterior a documentos frecuentemente consultados.

## Ejemplo de Uso

**Escenario:** El equipo de contabilidad necesita revisar el reporte de ventas por mes para el cierre del período, incluyendo una vista gráfica de la distribución mensual.

1. El usuario inicia sesión con su rol correspondiente y accede a la interfaz Retails de Solop ERP.
2. Desde el menú de reportes, selecciona el **modelo de reporte** y dentro de este elige el reporte específico requerido.
3. En el panel izquierdo, configura los parámetros: activa **Totales generales** y **Totales por líneas**.
4. Hace clic en el botón de generación. El sistema presenta el reporte con los datos del período actual.
5. El usuario verifica que los totales coincidan con el cierre esperado.
6. Accede a la pestaña de gráficos: el sistema muestra un gráfico de barras con los totales por mes.
7. El usuario cambia el tipo de gráfico a **líneas** para visualizar mejor la tendencia temporal y guarda el cambio.
8. Aplica un filtro por **grupo** para ver solo la información de interés.
9. El reporte y el gráfico se actualizan en tiempo real mostrando solo los datos filtrados.
10. Finalmente, el usuario prepara el reporte para impresión y lo archiva en la carpeta correspondiente.
